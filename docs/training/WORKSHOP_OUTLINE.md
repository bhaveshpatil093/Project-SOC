# ISRO ISTRAC SOC Platform — Analyst Training Workshop

This outline provides a structured 4-hour certification workshop designed to onboard Tier 1 Security Analysts onto the newly deployed AI-driven SOC Platform.

---

## Session 1: Introduction (30 min)
- **Why this platform exists**: The problem of Alert Fatigue. Transitioning from rigid rules to adaptive machine learning.
- **High-level architecture**: How logs from ISRO infrastructure become vectors, get scored by AI models, and turn into UI alerts.
- **Live demo**: Walkthrough of a full alert lifecycle — from raw Elasticsearch log to an SLM-assisted resolution.

## Session 2: Dashboard & Navigation (30 min)
- **Hands-on**: Log in using assigned training credentials and explore the default dashboard.
- **Understanding stat cards**: Translating "Active Incidents", "SLA Warnings", and threat level distribution charts.
- **Sidebar navigation**: Exploring the Alerts Queue, Investigation Chat, and Reporting tabs.

## Session 3: Alert Triage Fundamentals (45 min)
- **Threat Scores**: What a 0-100 score means, and how model *consensus* affects it.
- **Reading SHAP Explanations**: Understanding the "Why". (e.g. "Why did the Isolation Forest think this packet was bad?")
- **MITRE ATT&CK**: Deciphering the Tactics and Techniques badges.
- **Hands-on exercise**: Triage 5 pre-loaded sample alerts in the staging sandbox.

## Session 4: Using the AI Assistant (45 min)
- **Capabilities**: What the Small Language Model (SLM) *can* do (summarize logs, find historical context) and *cannot* do (guarantee 100% accuracy, block IPs).
- **Effective Prompting**: Asking clear, context-driven questions.
- **Hands-on**: Investigate a multi-stage incident strictly using the chat UI.
- **Guided Playbooks**: Leveraging the SLM to generate remediation steps.

## Session 5: Submitting Feedback (30 min)
- **The Learning Loop**: Explaining why clicking "False Positive" is the most important job of an analyst (Active Learning).
- **Hands-on**: Label 10 synthetic alerts as True Positive, False Positive, or Benign.
- **Suppression**: How the platform suppresses future noise based on analyst confidence scores.

## Session 6: Incidents & Correlation (30 min)
- **Alerts vs. Incidents**: An alert is a single event. An incident is a correlated story.
- **Attack Chain Timelines**: Reading chronological attack progression (e.g., Initial Access -> Lateral Movement -> Exfiltration).
- **Escalation**: When to escalate a multi-stage incident to Tier 2 / Incident Response.

## Session 7: Workbench Mode (Speed Training) (30 min)
- **Keyboard Shortcuts**: Ditching the mouse. Navigating the high-speed triage interface.
- **Hands-on Time Trial**: Trainees must triage 10 alerts in Workbench mode. The goal is an average triage time of under 60 seconds per alert.

## Session 8: Reports & Handover (20 min)
- **Generating shift reports**: Using the Reports tab to generate PDF handovers.
- **Best practices**: Writing clear, actionable notes for the incoming shift regarding active unresolved incidents.

## Hands-On Lab Exercises
*(Trainers will evaluate analysts on their handling of these 10 scenarios)*

1. **The Noisy Admin**: A known DB admin runs a massive `mysqldump` script. Expected Action: **False Positive**.
2. **The Suspicious Login**: 50 failed SSH attempts followed by 1 successful login from a foreign IP. Expected Action: **True Positive (Escalate)**.
3. **The Printer Beacon**: A misconfigured network printer constantly pinging an external NTP server. Expected Action: **Benign**.
4. **The Ghost Process**: `powershell.exe` spawns with a massive base64 encoded command string. Expected Action: **True Positive**.
5. **The Weekend Warrior**: An engineer logging in at 3 AM on a Sunday to check logs. Expected Action: **False Positive**.
6. **The Internal Scanner**: An authorized ISRO Qualys vulnerability scanner pinging all ports. Expected Action: **False Positive (Add Suppression Rule)**.
7. **The Data Hoarder**: A workstation zipping 50GB of files and attempting to FTP them outward. Expected Action: **True Positive (Escalate)**.
8. **The Broken Agent**: An endpoint repeatedly failing to authenticate due to an expired certificate. Expected Action: **Benign**.
9. **The Domain Admin Abuse**: A service account suddenly adding new users to the Domain Admins group. Expected Action: **True Positive (Critical Escalate)**.
10. **The Phishing Click**: A user clicks a malicious link, spawning a child process from Outlook. Expected Action: **True Positive**.

## Assessment Quiz
1. What does a Threat Score of 85 indicate?
2. If an alert is generated by an authorized internal IT backup script, how should it be labeled?
3. What is the difference between a False Positive and a Benign anomaly?
4. What does the "Consensus" badge mean on an alert?
5. How does clicking "False Positive" affect the machine learning models?
6. True or False: The SLM Assistant can automatically block IP addresses at the firewall.
7. What does the MITRE tactic "Initial Access" mean?
8. Name the keyboard shortcut to mark an alert as a True Positive in Workbench mode.
9. If you see a "Multi-stage" badge on an incident, what does that imply about the attacker?
10. What information is provided by a SHAP explanation?
11. When should you escalate an alert to the L2 team?
12. Why is it dangerous to ask the SLM "Is this a real attack?" without reviewing the evidence?
13. What is the maximum SLA response time for a Critical alert?
14. How do you find historical alerts related to the same IP address?
15. What should be included in a shift handover report?

---

## Certification Criteria
- [ ] Complete all 8 hands-on sessions.
- [ ] Score 80%+ (12/15) on the assessment quiz.
- [ ] Successfully triage 5 lab scenarios with correct TP/FP/Benign classifications and provide valid reasoning.
- [ ] Demonstrate Workbench mode proficiency (average < 60s triage time on the lab set).
