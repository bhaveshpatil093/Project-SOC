# ISRO ISTRAC SOC Platform — Stakeholder Demo Script

**Audience:** ISRO project supervisors, SOC team leadership, technical reviewers
**Duration:** 20-25 minutes
**Goal:** Demonstrate the complete value chain from raw log to resolved threat

## Pre-Demo Setup (do this before stakeholders arrive)
1. Ensure all services are running: `make docker-up && docker compose ps`
2. Pre-seed realistic demo data: `python backend/scripts/seed_demo_scenario.py`
   *(Creates a believable multi-stage attack scenario with realistic timestamps)*
3. Open browser tabs: Dashboard (`http://localhost:5173`), Kibana (backup), Grafana (backup)
4. Have this script open on a second screen

---

## Part 1: The Problem (2 min — no platform, just talk)
"SOC L1 engineers face massive alert fatigue — hundreds of daily alerts are generated, and the vast majority are false positives. Investigating a single alert manually requires parsing raw logs and can take 15-20 minutes. 

This AI platform was built to solve three core problems: 
1. Drastically reduce noise via ML anomaly detection.
2. Explain *why* something was flagged so analysts don't operate blindly.
3. Provide an AI assistant (SLM) to drastically accelerate the investigation phase."

## Anticipated Questions & Prepared Answers

**Q: "How accurate is it really?"**
A: "Our baseline testing shows a False Positive reduction of over 40% compared to static rules, with Precision climbing steadily as the Active Learning loop ingests analyst feedback."

**Q: "What if the AI gives a wrong answer?"**
A: "The AI is explicitly an *assistant*, not a replacement for human judgment. Every AI response includes direct citations to the raw logs. The human analyst always makes the final triage call. The system just makes them vastly faster."

**Q: "What happens if Elasticsearch goes down?"**
A: "The platform has a localized Redis cache to queue incoming alerts, and the infrastructure is built with strict Docker resource limits and automated health checks to prevent cascading failures."

**Q: "Can this scale to handle more analysts or more log volume?"**
A: "Yes. The ingestion engine is fully asynchronous and decoupled from the ML inference loop. If log volumes double, we simply scale the Elasticsearch cluster horizontally. We have documented the exact CPU/RAM scaling requirements in our Performance Baseline manual."
